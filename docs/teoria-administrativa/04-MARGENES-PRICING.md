## 1. Introducción a Pricing y Márgenes

El pricing (fijación de precios) y la gestión de márgenes son dos de las decisiones más críticas en cualquier negocio. Un cambio de 1% en el precio puede impactar la rentabilidad mucho más que un cambio del 1% en costos o volumen.

### Relación Precio-Margen-Volumen

```
Revenue = Price × Volume
Cost = Fixed Costs + (Variable Cost per Unit × Volume)
Profit = Revenue - Cost
Margin = (Profit / Revenue) × 100
```

### Por Qué el Pricing es Crítico

**Impacto en Profitabilidad:**

Ejemplo: Empresa con revenue de $1M y profit de $100K (10% margin)

| Mejora del 1% en: | Impacto en Profit | Incremento % |
|-------------------|-------------------|--------------|
| Precio (sin perder volumen) | +$10,000 | +10% |
| Volumen (sin cambiar precio) | +$1,000 | +1% |
| Costos variables | +$4,500 | +4.5% |
| Costos fijos | +$5,000 | +5% |

**Conclusión:** El precio tiene el mayor impacto leverage en profitabilidad.

### Objetivos del Pricing

1. **Maximizar Profitabilidad** (no siempre revenue)
2. **Capturar Valor** creado para el cliente
3. **Posicionamiento** en el mercado
4. **Competitividad** sostenible
5. **Cashflow** adecuado
6. **Market Share** según estrategia

## 2. Tipos de Márgenes

### 2.1 Gross Margin (Margen Bruto)

El margen bruto mide la rentabilidad **después de costos directos** pero **antes de overhead**.

#### Fórmula

```
Gross Profit = Revenue - Cost of Goods Sold (COGS)
Gross Margin % = (Gross Profit / Revenue) × 100
```

**COGS incluye:**
- Materiales directos
- Mano de obra directa
- Costos directos de producción

**COGS NO incluye:**
- Overhead (renta, admin, marketing)
- Intereses
- Impuestos

#### Ejemplo: Retail Store

**Income Statement:**
```
Revenue (Sales)                    $500,000
COGS (Cost of merchandise)         $300,000
────────────────────────────────────────────
Gross Profit                       $200,000
Gross Margin %                     40%

Operating Expenses:
  Rent                             $40,000
  Salaries                         $80,000
  Marketing                        $20,000
  Utilities                        $10,000
  Other                            $15,000
  Total Operating Expenses         $165,000
────────────────────────────────────────────
Operating Profit (EBIT)            $35,000
```

```
Gross Margin = ($200,000 / $500,000) × 100 = 40%
```

#### Uso del Gross Margin

- **Comparar productos/categorías** dentro de la misma empresa
- **Benchmarking** contra competidores
- **Evaluar poder de pricing**
- **Analizar eficiencia de producción/compras**

#### Gross Margin Benchmarks

| Industria | Typical Gross Margin |
|-----------|---------------------|
| Software/SaaS | 70-90% |
| Professional Services | 50-70% |
| Retail (apparel) | 40-55% |
| Retail (grocery) | 20-30% |
| Manufacturing | 25-40% |
| Restaurants | 60-70% (food only) |
| Construction | 15-25% |

### 2.2 Net Margin (Margen Neto)

El margen neto mide la rentabilidad **final** después de TODOS los gastos.

#### Fórmula

```
Net Profit = Revenue - All Expenses (including COGS, overhead, interest, taxes)
Net Margin % = (Net Profit / Revenue) × 100
```

#### Ejemplo (continuando del anterior)

```
Operating Profit (EBIT)            $35,000
Interest Expense                   ($3,000)
────────────────────────────────────────────
Profit Before Tax (EBT)            $32,000
Income Tax (25%)                   ($8,000)
────────────────────────────────────────────
Net Profit                         $24,000

Net Margin = ($24,000 / $500,000) × 100 = 4.8%
```

#### Interpretación

- **Net Margin** es el "bottom line"
- Refleja **eficiencia operacional total**
- Incluye impacto de estructura de capital (intereses)
- Varía significativamente por industria

#### Net Margin Benchmarks

| Industria | Typical Net Margin |
|-----------|-------------------|
| Software/SaaS | 15-25% |
| Retail | 2-8% |
| Restaurants | 3-10% |
| Grocery stores | 1-3% |
| Manufacturing | 5-15% |
| Construction | 3-7% |
| Professional Services | 10-20% |

### 2.3 Contribution Margin (Margen de Contribución)

El contribution margin mide cuánto contribuye cada unidad vendida a cubrir **costos fijos y generar profit**.

#### Fórmula

```
Contribution Margin per Unit = Price - Variable Cost per Unit
Contribution Margin % = (CM per Unit / Price) × 100
```

O en términos totales:
```
Total Contribution Margin = Total Revenue - Total Variable Costs
CM Ratio = Total CM / Total Revenue
```

#### Diferencia con Gross Margin

**Gross Margin:**
- Revenue - COGS
- COGS puede incluir algunos costos fijos de producción (en absorption costing)

**Contribution Margin:**
- Revenue - ONLY variable costs
- Separa estrictamente fijo vs variable

#### Ejemplo: Manufacturing Company

**Producto: Widget**

| Item | Amount |
|------|--------|
| Selling Price | $100 |
| Variable Costs: | |
| - Materials | $30 |
| - Direct labor | $20 |
| - Variable overhead | $10 |
| **Total Variable Cost** | **$60** |
| **Contribution Margin** | **$40** |
| **CM %** | **40%** |

**Si se venden 1,000 units:**

```
Total Revenue = 1,000 × $100 = $100,000
Total Variable Costs = 1,000 × $60 = $60,000
Total Contribution Margin = $40,000

Fixed Costs = $25,000 (rent, salaries, etc.)

Profit = CM - Fixed Costs = $40,000 - $25,000 = $15,000
```

#### Uso del Contribution Margin

- **Break-even analysis**
- **Decisiones de pricing** a corto plazo
- **Análisis de producto mix**
- **Make-or-buy decisions**
- **Special order decisions**

### 2.4 Operating Margin

Mide rentabilidad de operaciones core, **excluyendo intereses e impuestos**.

```
Operating Profit (EBIT) = Revenue - COGS - Operating Expenses
Operating Margin % = (EBIT / Revenue) × 100
```

**Ventaja:** Permite comparar empresas con diferentes estructuras de capital y tasas impositivas.

#### Ejemplo

```
Revenue:                  $1,000,000
COGS:                      ($600,000)
Gross Profit:              $400,000
Operating Expenses:        ($280,000)
────────────────────────────────────
Operating Profit (EBIT):   $120,000

Operating Margin = ($120,000 / $1,000,000) × 100 = 12%
```

### 2.5 EBITDA Margin

Similar a operating margin pero **agrega back depreciation y amortization**.

```
EBITDA = EBIT + Depreciation + Amortization
EBITDA Margin % = (EBITDA / Revenue) × 100
```

**Uso:**
- Comparar empresas con diferentes niveles de capital assets
- Valuation (multiples de EBITDA)
- Aproximación de cash flow operativo

#### Ejemplo

```
Operating Profit (EBIT):   $120,000
Depreciation:              $30,000
Amortization:              $10,000
────────────────────────────────────
EBITDA:                    $160,000

EBITDA Margin = ($160,000 / $1,000,000) × 100 = 16%
```

## 3. Markup vs Margin

Esta es una de las confusiones más comunes en pricing. **Markup y margin NO son lo mismo.**

### 3.1 Definiciones y Diferencias

#### Markup

**Base:** Costo

```
Markup % = (Price - Cost) / Cost × 100
```

**Ejemplo:**
- Cost: $60
- Price: $100

```
Markup = ($100 - $60) / $60 × 100 = 66.67%
```

**Interpretación:** El precio es 66.67% MÁS que el costo.

#### Margin

**Base:** Precio de venta

```
Margin % = (Price - Cost) / Price × 100
```

**Mismo ejemplo:**
- Cost: $60
- Price: $100

```
Margin = ($100 - $60) / $100 × 100 = 40%
```

**Interpretación:** El margen es 40% del precio de venta.

### 3.2 Fórmulas de Conversión

#### Convertir Markup a Margin

```
Margin % = Markup % / (1 + Markup %)
```

**Ejemplo:**
```
Markup = 66.67% = 0.6667
Margin = 0.6667 / (1 + 0.6667) = 0.6667 / 1.6667 = 0.40 = 40%
```

#### Convertir Margin a Markup

```
Markup % = Margin % / (1 - Margin %)
```

**Ejemplo:**
```
Margin = 40% = 0.40
Markup = 0.40 / (1 - 0.40) = 0.40 / 0.60 = 0.6667 = 66.67%
```

#### Calcular Precio desde Costo y Markup

```
Price = Cost × (1 + Markup %)
```

**Ejemplo:**
```
Cost = $60, Markup = 66.67%
Price = $60 × (1 + 0.6667) = $60 × 1.6667 = $100
```

#### Calcular Precio desde Costo y Margin

```
Price = Cost / (1 - Margin %)
```

**Ejemplo:**
```
Cost = $60, Target Margin = 40%
Price = $60 / (1 - 0.40) = $60 / 0.60 = $100
```

#### Ejemplo de Error Común

```
Margin = ($70 - $50) / $70 = 28.6%
```

Verificación:
```
Margin = ($83.33 - $50) / $83.33 = 40% ✓
```

## 4. Break-Even Analysis

### 4.1 Conceptos Fundamentales

**Break-Even Point (BEP)** es el nivel de ventas donde **Total Revenue = Total Costs** (profit = $0).

#### Componentes

**Fixed Costs (FC):**
- Costos que NO cambian con volumen de producción
- Ejemplos: Renta, salarios fijos, insurance, depreciation

**Variable Costs (VC):**
- Costos que cambian directamente con volumen
- Expresado típicamente como "costo variable por unidad"
- Ejemplos: Materiales, commissions, packaging

**Total Cost:**
```
Total Cost = Fixed Costs + (Variable Cost per Unit × Quantity)
TC = FC + (VC × Q)
```

**Total Revenue:**
```
Total Revenue = Price × Quantity
TR = P × Q
```

### 4.2 Break-Even Point en Unidades

```
BEP (units) = Fixed Costs / (Price - Variable Cost per Unit)
```

O usando contribution margin:
```
BEP (units) = Fixed Costs / Contribution Margin per Unit
```

#### Ejemplo: Coffee Shop

**Datos:**
- Selling price per coffee: $5.00
- Variable cost per coffee: $2.00
- Fixed costs (monthly): $15,000 (rent, salaries, utilities)

**Cálculo:**

```
Contribution Margin per Unit = $5.00 - $2.00 = $3.00

BEP = $15,000 / $3.00 = 5,000 coffees/month
```

**Verificación:**
```
Revenue at BEP = 5,000 × $5.00 = $25,000
Variable Costs = 5,000 × $2.00 = $10,000
Fixed Costs = $15,000
Total Costs = $10,000 + $15,000 = $25,000

Profit = $25,000 - $25,000 = $0 ✓
```

**Interpretación:** El coffee shop debe vender 5,000 cafés al mes para cubrir todos los costos.

Si vende más de 5,000: Genera profit  
Si vende menos de 5,000: Tiene pérdida

### 4.3 Break-Even Point en Dólares

```
BEP ($) = Fixed Costs / Contribution Margin Ratio

Donde: CM Ratio = (Price - Variable Cost) / Price
```

#### Ejemplo (mismo coffee shop)

```
CM Ratio = ($5.00 - $2.00) / $5.00 = 0.60 = 60%

BEP ($) = $15,000 / 0.60 = $25,000 in sales
```

**Verificación:**
```
Units at $25,000 sales = $25,000 / $5 = 5,000 units ✓
```

#### Cuando Usar BEP en Dólares

- Múltiples productos con diferentes precios
- Análisis de revenue targets
- Industrias donde se mide por $ sales más que unidades

### 4.4 Margin of Safety

Mide cuánto pueden caer las ventas antes de llegar a break-even (pérdida).

```
Margin of Safety (units) = Actual Sales - Break-Even Sales
Margin of Safety % = (MoS / Actual Sales) × 100
```

#### Ejemplo

Coffee shop del ejemplo vende **8,000 coffees/month** (actual).

```
BEP = 5,000 coffees
Actual = 8,000 coffees

Margin of Safety = 8,000 - 5,000 = 3,000 coffees
MoS % = (3,000 / 8,000) × 100 = 37.5%
```

**Interpretación:** Las ventas pueden caer 37.5% antes de llegar a break-even.

**Benchmarks:**
- MoS > 30%: Saludable
- MoS 15-30%: Aceptable
- MoS < 15%: Riesgoso

### 4.5 Target Profit Analysis

¿Cuántas unidades necesito vender para alcanzar un profit target?

```
Required Units = (Fixed Costs + Target Profit) / Contribution Margin per Unit
```

#### Ejemplo

Coffee shop quiere $9,000 de profit mensual.

```
FC = $15,000
Target Profit = $9,000
CM per unit = $3.00

Required Units = ($15,000 + $9,000) / $3.00 = 8,000 coffees
```

**Verificación:**
```
Revenue = 8,000 × $5 = $40,000
Variable Costs = 8,000 × $2 = $16,000
Fixed Costs = $15,000
Total Costs = $31,000

Profit = $40,000 - $31,000 = $9,000 ✓
```

#### Target Profit en Revenue

```
Required Revenue = (Fixed Costs + Target Profit) / CM Ratio
```

**Ejemplo:**
```
Required Revenue = ($15,000 + $9,000) / 0.60 = $40,000
```

## 5. Cost-Volume-Profit (CVP) Analysis

CVP analiza cómo cambios en **costos, volumen y precio** afectan el profit.

### 5.1 CVP Formula

```
Profit = (Price × Quantity) - (Variable Cost per Unit × Quantity) - Fixed Costs
Profit = (P × Q) - (VC × Q) - FC
Profit = Q × (P - VC) - FC
Profit = Q × CM - FC
```

#### Ejemplo Multi-Escenario

**Base case:**
- Price: $50
- Variable cost: $30
- Fixed costs: $100,000
- Volume: 8,000 units

```
Profit = 8,000 × ($50 - $30) - $100,000
Profit = 8,000 × $20 - $100,000
Profit = $160,000 - $100,000 = $60,000
```

**Scenario 1: Increase price 10% (to $55)**
```
Profit = 8,000 × ($55 - $30) - $100,000
Profit = 8,000 × $25 - $100,000 = $100,000
Change: +$40,000 (+66.7%)
```

**Scenario 2: Reduce variable cost 10% (to $27)**
```
Profit = 8,000 × ($50 - $27) - $100,000
Profit = 8,000 × $23 - $100,000 = $84,000
Change: +$24,000 (+40%)
```

**Scenario 3: Increase volume 10% (to 8,800)**
```
Profit = 8,800 × ($50 - $30) - $100,000
Profit = 8,800 × $20 - $100,000 = $76,000
Change: +$16,000 (+26.7%)
```

**Conclusion:** Price changes have the biggest impact on profit.

### 5.2 Operating Leverage

Mide qué tan sensible es el profit a cambios en volumen.

```
Degree of Operating Leverage (DOL) = Contribution Margin / Profit
```

**Interpretación:** Por cada 1% de cambio en sales, el profit cambia DOL%.

#### Ejemplo

```
Total CM = $160,000
Profit = $60,000

DOL = $160,000 / $60,000 = 2.67
```

**Significado:** Si las ventas aumentan 10%, el profit aumentará 10% × 2.67 = 26.7%.

**Verificación:**
- New volume: 8,000 × 1.10 = 8,800 units
- New profit: 8,800 × $20 - $100,000 = $76,000
- Change: ($76,000 - $60,000) / $60,000 = 26.7% ✓

#### High vs Low Operating Leverage

**High Operating Leverage** (FC altos, VC bajos):
- Ejemplo: Software, airlines
- Profit muy sensible a cambios en volumen
- Alto riesgo, alto reward

**Low Operating Leverage** (FC bajos, VC altos):
- Ejemplo: Retail, consulting
- Profit menos sensible a volumen
- Bajo riesgo, pero menor upside

### 5.3 Sensitivity Analysis

Analizar múltiples variables cambiando simultáneamente.

#### Ejemplo: Sensitivity Table

**Base:** Price $50, VC $30, Volume 8,000, FC $100,000 → Profit $60,000

**Cambio en Price y Volume:**

| Price ↓ Volume → | 7,000 | 8,000 | 9,000 |
|------------------|-------|-------|-------|
| **$45** | $5,000 | $20,000 | $35,000 |
| **$50** (base) | $40,000 | $60,000 | $80,000 |
| **$55** | $75,000 | $100,000 | $125,000 |

**Insight:** Un aumento de precio a $55 puede compensar una caída de volumen de 8,000 a 7,000 y aún aumentar profit.

## 6. Pricing Strategies

### 6.1 Cost-Plus Pricing

Establecer precio sumando un markup al costo.

```
Price = Cost × (1 + Markup %)
```

O para target margin:
```
Price = Cost / (1 - Target Margin %)
```

#### Ejemplo

**Producto:**
- Manufacturing cost: $40
- Target margin: 35%

```
Price = $40 / (1 - 0.35) = $40 / 0.65 = $61.54
Rounded: $59.99 o $62.99
```

#### Ventajas

✅ Simple y fácil de implementar  
✅ Asegura cobertura de costos  
✅ Pricing consistente  
✅ Justificable internamente

#### Desventajas

❌ Ignora valor para el cliente  
❌ Ignora competencia  
❌ Puede dejar dinero en la mesa  
❌ No incentiva eficiencia (costos altos → precios altos)

#### Cuándo Usar

- Productos commodity
- Contratos de gobierno (cost-plus contracts)
- Industrias reguladas
- Cuando el mercado es muy sensible al precio

### 6.2 Competitive Pricing

Establecer precio basándose en la competencia.

#### Estrategias

**1. At-Market Pricing**
- Precio igual o muy similar a competidores
- Diferenciación por otros factores (servicio, calidad)

**2. Premium Pricing**
- Precio superior a competencia (+10-50%)
- Requiere diferenciación percibida

**3. Discount Pricing**
- Precio inferior a competencia (-10-30%)
- Estrategia de volumen o penetración

#### Proceso

1. **Identificar competidores** relevantes
2. **Research pricing** (website, mystery shopping, industry reports)
3. **Analizar posicionamiento** de cada competidor
4. **Determinar tu posición** estratégica
5. **Set price** relativo a competencia
6. **Monitor** cambios del mercado

#### Ejemplo: E-commerce Product

**Competitive Analysis:**

| Competitor | Price | Rating | Shipping |
|------------|-------|--------|----------|
| Amazon | $49.99 | 4.5★ | Free (Prime) |
| Walmart | $44.99 | 4.2★ | Free >$35 |
| Target | $54.99 | 4.3★ | Free >$35 |
| Best Buy | $52.99 | 4.4★ | Free >$35 |
| **Average** | **$50.74** | **4.35★** | - |

**Your Product:**
- Quality: Similar to Amazon
- Brand: Less known
- Service: Excellent (4.7★ rating)

**Strategy:** Premium to average but discount to Amazon
```
Target Price: $47.99
- Below Amazon (-$2.00)
- Above average (-$2.75 = -5.4%)
- Premium value story: "Same quality, better service, better price"
```

#### Ventajas

✅ Market-oriented  
✅ Reduce riesgo de mispricing  
✅ Responde a realidad del mercado

#### Desventajas

❌ Puede iniciar price wars  
❌ Ignora tu estructura de costos  
❌ Puede no ser optimal para tu profit

### 6.3 Value-Based Pricing

Precio basado en el **valor percibido** por el cliente.

#### Fórmula Conceptual

```
Price = % del Valor Creado para el Cliente
```

#### Proceso

**Step 1: Identificar el valor**
- ¿Qué problema resuelve?
- ¿Cuánto ahorra/genera para el cliente?
- ¿Cuál es la alternativa?

**Step 2: Cuantificar**
- Traducir beneficios a $
- Calcular ROI para el cliente

**Step 3: Determinar % apropiado**
- Típicamente 10-40% del valor del primer año
- O 3-10% del valor total (multi-year)

**Step 4: Validar contra costos**
- Asegurar que el precio cubre costos + margen mínimo

#### Ejemplo: Software de Productividad

**Cliente:** Empresa con 50 empleados

**Valor creado:**
- Ahorra 30 min/día por empleado
- Salario promedio: $60k/year = $30/hour loaded cost
- Días laborables: 250/year

```
Ahorro diario = 50 empleados × 0.5 hrs × $30/hr = $750/day
Ahorro anual = $750 × 250 = $187,500/year
```

**Value-Based Price:**
- Cobrar 20% del valor del primer año
```
Annual Price = $187,500 × 0.20 = $37,500/year
Monthly Price = $37,500 / 12 = $3,125/month
```

**Validación:**

Cost to provide service: $10,000/year
```
Gross Margin = ($37,500 - $10,000) / $37,500 = 73.3% ✓ Excelente
```

Client ROI:
```
Client pays: $37,500
Client saves: $187,500
ROI: ($187,500 - $37,500) / $37,500 = 400% ✓ Compelling
```

#### Ventajas

✅ Maximiza revenue potential  
✅ Captura el valor creado  
✅ Win-win (cliente tiene ROI positivo)  
✅ Permite pricing premium

#### Desventajas

❌ Difícil cuantificar valor (productos intangibles)  
❌ Requiere deep discovery con cliente  
❌ Valor puede variar por cliente  
❌ Más complejo de implementar

### 6.4 Penetration Pricing

Precio inicial **bajo** para ganar market share rápidamente.

#### Estrategia

```
Initial Price << Market Price
→ Gain customers quickly
→ Build scale
→ Increase price later (or maintain with economies of scale)
```

#### Ejemplo: Streaming Service

**Market:**
- Netflix: $15.99/month
- Disney+: $10.99/month
- Hulu: $14.99/month

**New entrant strategy:**
```
Launch price: $5.99/month (60% below market)
Goal: Reach 1M subscribers in first year
After 1 year: Increase to $9.99/month
```

#### Cuándo Usar

✅ Entering a competitive market  
✅ High price elasticity  
✅ Economies of scale exist  
✅ Network effects (más usuarios = más valor)  
✅ Can afford initial losses

#### Riesgos

❌ Race to the bottom  
❌ Hard to raise prices later  
❌ Attracts price-sensitive customers (high churn)  
❌ May signal low quality  
❌ Puede no ser sustainable

### 6.5 Skimming Pricing

Precio inicial **alto**, luego bajar gradualmente.

#### Estrategia

```
Launch: High Price (early adopters)
→ Month 3-6: Reduce 10-20%
→ Month 6-12: Reduce otro 10-20%
→ Mature: Market price
```

#### Ejemplo: Tecnología (iPhone-like)

```
Launch (Month 0): $999 (flagship model)
Month 3: $949 (-5%)
Month 6: $899 (-10%)
Month 9: $849 (-15%)
Month 12: $799 (-20%)
Next generation launch: New model at $999, old model to $699
```

#### Cuándo Usar

✅ Innovative/unique product  
✅ Limited competition initially  
✅ Inelastic demand (early adopters)  
✅ High production costs initially (scale not yet achieved)  
✅ Technology products

#### Ventajas

✅ Maximizes revenue from early adopters  
✅ Recovers R&D costs quickly  
✅ Creates perception of premium/quality  
✅ Easier to lower price than raise it

## 7. Price Elasticity

### 7.1 Concepto de Elasticidad

**Price Elasticity of Demand (PED)** mide qué tan sensible es la demanda a cambios en precio.

```
PED = (% Change in Quantity Demanded) / (% Change in Price)
```

#### Ejemplo Básico

Producto tiene precio de $10 y vende 1,000 units.

Precio aumenta a $11 (+10%), ventas caen a 900 units (-10%).

```
PED = (-10%) / (+10%) = -1.0
```

**Nota:** PED típicamente es negativo (precio ↑ → demanda ↓).

### 7.2 Cálculo de Elasticidad

#### Método Mid-Point (más preciso)

```
PED = ((Q2 - Q1) / ((Q2 + Q1)/2)) / ((P2 - P1) / ((P2 + P1)/2))
```

**Ejemplo:**
- P1 = $10, Q1 = 1,000
- P2 = $11, Q2 = 900

```
ΔQ = 900 - 1,000 = -100
Avg Q = (900 + 1,000) / 2 = 950
%ΔQ = -100 / 950 = -10.53%

ΔP = 11 - 10 = 1
Avg P = (11 + 10) / 2 = 10.5
%ΔP = 1 / 10.5 = 9.52%

PED = -10.53% / 9.52% = -1.11
```

### 7.3 Elastic vs Inelastic Demand

#### Clasificación

| PED Value | Classification | Meaning |
|-----------|---------------|---------|
| PED = 0 | Perfectly Inelastic | Demand no cambia con precio |
| 0 < \|PED\| < 1 | Inelastic | Demand poco sensible a precio |
| \|PED\| = 1 | Unit Elastic | % cambio demand = % cambio precio |
| \|PED\| > 1 | Elastic | Demand muy sensible a precio |
| \|PED\| = ∞ | Perfectly Elastic | Cualquier cambio precio → demand a 0 |

#### Inelastic Demand (\|PED\| < 1)

**Características:**
- Productos esenciales/necesarios
- Pocas alternativas
- Pequeña parte del presupuesto
- Adicción/hábito

**Ejemplos:**
- Gasolina (\|PED\| ≈ 0.2-0.5)
- Medicamentos (especialmente sin genéricos)
- Sal, azúcar
- Utilities (electricidad, agua)

**Implicación para pricing:**
✅ **Aumentar precio AUMENTA revenue** (demand cae poco)

**Ejemplo:**
```
Base: P=$10, Q=1,000, PED=-0.5
Increase price 10% to $11

Expected demand change = -0.5 × 10% = -5%
New Q = 1,000 × (1 - 0.05) = 950

Old Revenue: $10 × 1,000 = $10,000
New Revenue: $11 × 950 = $10,450
Change: +$450 (+4.5%) ✅
```

#### Elastic Demand (\|PED\| > 1)

**Características:**
- Productos de lujo/no esenciales
- Muchas alternativas/substitutes
- Gran parte del presupuesto
- Brand switching fácil

**Ejemplos:**
- Ropa de marca (\|PED\| ≈ 1.5-2.5)
- Restaurantes
- Entretenimiento
- Viajes leisure

**Implicación para pricing:**
⚠️ **Aumentar precio REDUCE revenue** (demand cae mucho)

**Ejemplo:**
```
Base: P=$50, Q=1,000, PED=-2.0
Increase price 10% to $55

Expected demand change = -2.0 × 10% = -20%
New Q = 1,000 × (1 - 0.20) = 800

Old Revenue: $50 × 1,000 = $50,000
New Revenue: $55 × 800 = $44,000
Change: -$6,000 (-12%) ❌
```

**Strategy:** Para productos elásticos, **reducir precio puede aumentar revenue**.

#### Factores que Afectan Elasticidad

1. **Availability of Substitutes**
   - Más substitutes → Más elástico
   - Ejemplo: Marca específica de cereales (muy elástico) vs cereales en general (menos elástico)

2. **Necessity vs Luxury**
   - Necesidad → Inelástico
   - Lujo → Elástico

3. **Proportion of Income**
   - Pequeña compra → Inelástico (goma de mascar)
   - Gran compra → Elástico (auto)

4. **Time Horizon**
   - Corto plazo → Más inelástico
   - Largo plazo → Más elástico (tiempo para encontrar alternativas)

5. **Brand Loyalty**
   - Alta lealtad → Menos elástico
   - Baja lealtad → Más elástico

## 8. Optimal Pricing

### 8.1 Maximización de Profit

El precio optimal maximiza **profit**, no necesariamente revenue o market share.

#### Fórmula Teórica

En economía perfecta con demand curve conocida:

```
Optimal Price = (Variable Cost per Unit) / (1 + (1/PED))
```

**Nota:** Esta fórmula asume conocimiento perfecto de PED, lo cual en realidad requiere testing.

#### Ejemplo Simplificado

```
Variable Cost = $20/unit
PED = -2.0

Optimal Price = $20 / (1 + (1/-2)) = $20 / (1 - 0.5) = $20 / 0.5 = $40
```

#### Enfoque Práctico: Testing

**Proceso:**

1. **Establish baseline**
   - Current price, volume, profit

2. **Test price changes**
   - A/B test diferentes price points
   - Por segmento, geografía, o temporalmente

3. **Measure impact**
   - No solo volume, sino profit total

4. **Iterate**
   - Move toward profit-maximizing price

### 8.2 Revenue Optimization

A veces el objetivo es maximizar **revenue** (no profit).

**Cuándo:**
- Startups buscando scale
- Market share prioritario
- Plataformas con network effects

#### Revenue Maximization Point

```
Revenue = Price × Quantity

Revenue is maximized when: PED = -1 (unit elastic)
```

**Intuición:**
- Si \|PED\| > 1: Reducir precio aumenta revenue (demand sube más)
- Si \|PED\| < 1: Aumentar precio aumenta revenue (demand baja poco)
- At \|PED\| = 1: Revenue está en máximo

#### Ejemplo

**Demand curve:** Q = 1,000 - 10P

```
Revenue = P × Q = P × (1,000 - 10P) = 1,000P - 10P²

To maximize, derivative = 0:
dR/dP = 1,000 - 20P = 0
P = 50

At P=$50:
Q = 1,000 - 10(50) = 500
Revenue = $50 × 500 = $25,000 (máximo)
```

### 8.3 Market Share vs Profitability

Trade-off común: Precio bajo gana market share pero reduce profit margin.

#### Ejemplo Comparativo

**Scenario A: Low Price (Market Share Strategy)**
```
Price: $30
Volume: 10,000 units
Revenue: $300,000
Variable Cost: $20/unit = $200,000
Fixed Cost: $50,000
Profit: $50,000 (16.7% margin)
Market Share: 25%
```

**Scenario B: High Price (Profitability Strategy)**
```
Price: $45
Volume: 6,000 units
Revenue: $270,000
Variable Cost: $20/unit = $120,000
Fixed Cost: $50,000
Profit: $100,000 (37% margin)
Market Share: 15%
```

**Analysis:**
- Scenario B: Menos revenue (-10%), pero DOBLE profit
- Scenario B: Menos market share (-10 puntos)

**Decision factors:**
- Stage of company (startup vs mature)
- Strategic goals (growth vs profitability)
- Competitive dynamics
- Investor expectations

## 9. Pricing Across Industries

### 9.1 Retail Pricing

#### Keystone Pricing

Método tradicional: **2x cost**

```
Wholesale Cost = $50
Retail Price = $50 × 2 = $100
```

Esto da 50% margin:
```
Margin = ($100 - $50) / $100 = 50%
```

**Cuándo usar:** Productos estándar, middle-market.

#### Multiple Pricing

Diferentes multipliers por categoría:

| Category | Multiplier | Margin % |
|----------|-----------|----------|
| Jewelry | 3.0x | 66.7% |
| Electronics | 1.2x | 16.7% |
| Apparel | 2.5x | 60.0% |
| Groceries | 1.3x | 23.1% |
| Furniture | 2.2x | 54.5% |

**Razón:** Varia por:
- Holding costs (inventory turnover)
- Markdowns esperados
- Competencia
- Customer expectations

#### Markdown Strategy

```
Original Price: $100 (50% margin)
Week 4: -20% → $80 (37.5% margin)
Week 8: -30% → $70 (28.6% margin)
Week 12: -50% → $50 (0% margin, at cost)
```

**Planeación:** Factor expected markdowns en initial price.

```
If expect 30% of inventory sells at -20% discount:
Blended margin = (70% × 50%) + (30% × 37.5%) = 35% + 11.25% = 46.25%
```

### 9.2 Manufacturing Pricing

#### Cost-Plus con Full Absorption

```
Direct Materials: $30
Direct Labor: $20
Variable Overhead: $10
Fixed Overhead (allocated): $15
───────────────────────────
Full Cost: $75

Target Margin: 30%
Price = $75 / (1 - 0.30) = $107.14
```

#### Price Floors

**Absolute Floor:** Variable cost
- Below this = losing money on each unit

**Strategic Floor:** Full cost
- Below this = not covering allocated overhead

**Example:**
```
Variable Cost = $60
Full Cost = $75
Normal Price = $107

Special Order Analysis:
If offered $70/unit for large order (excess capacity):
✅ Accept: $70 > $60 (contributes $10/unit to fixed costs)
❌ If at capacity: Reject (below full cost and normal price)
```

### 9.4 SaaS/Subscription Pricing

#### Modelos Comunes

**1. Per-User Pricing**
```
$15/user/month
Team of 10 = $150/month
```

**Ventajas:** Simple, scales con customer growth  
**Desventajas:** Incentiva account sharing

**2. Tiered Pricing**
```

```

**3. Usage-Based Pricing**
```

+ $0.10 per API call
+ $5 per GB storage
```

**Ejemplo:** AWS, Twilio

**4. Freemium**
```
Free: Basic features, 100 transactions/month

```

#### Pricing Psychology

**Good-Better-Best:**
```
BASIC          PROFESSIONAL       ENTERPRISE
$19/mo         $49/mo ⭐          $149/mo
(Anchor low)   (Target tier)      (Upsell)
```

**Mayoría de clientes** eligen middle tier.

#### Metrics clave

**Customer Lifetime Value (LTV):**
```
LTV = (Avg Monthly Revenue per Customer × Gross Margin %) / Monthly Churn Rate
```

**Example:**
```
Avg MRR = $100
Gross Margin = 80%
Monthly Churn = 5%

LTV = ($100 × 0.80) / 0.05 = $80 / 0.05 = $1,600
```

```
CAC = Total Sales & Marketing Spend / New Customers Acquired
```

**LTV:CAC Ratio:**
- Target: **3:1 or higher**
- Below 3:1 = not sustainable
- Above 5:1 = under-investing in growth

## 10. Discounting Strategies

### 10.1 Types of Discounts

#### Volume Discounts

```
1-10 units: $100 each
11-50 units: $90 each (10% off)
51+ units: $80 each (20% off)
```

**Purpose:** Incentivize larger purchases

#### Early Payment Discounts

```
Terms: 2/10 Net 30
= 2% discount if paid within 10 days, otherwise full payment due in 30 days
```

**Analysis:**

If invoice = $10,000:
- Pay in 10 days: $10,000 × 0.98 = $9,800
- Pay in 30 days: $10,000

Saving $200 for paying 20 days earlier.

**Annualized rate:**
```
($200 / $9,800) × (365 / 20) = 0.0204 × 18.25 = 37.2% annual rate
```

**Conclusion:** Good deal for customer to take discount!

#### Seasonal Discounts

```
Winter coats:
- September (pre-season): Full price $200
- December (peak): Full price $200
- January (post-season): 30% off = $140
- February: 50% off = $100
```

#### Bundle Discounts

Ya cubierto en sección 6.7.

### 10.2 Impact Analysis

#### Revenue Impact

**Example:**
- Normal: 1,000 units @ $100 = $100,000
- With 20% discount: 1,300 units @ $80 = $104,000

```
Volume increase needed to maintain revenue:
Let X = new volume
X × $80 = $100,000
X = 1,250 units

Required increase: (1,250 - 1,000) / 1,000 = 25%
```

**Conclusion:** Need 25% more volume just to maintain same revenue.

#### Profit Impact

```
Cost per unit: $60

Normal:
Revenue: 1,000 × $100 = $100,000
Cost: 1,000 × $60 = $60,000
Profit: $40,000

With 20% discount, 1,300 units:
Revenue: 1,300 × $80 = $104,000
Cost: 1,300 × $60 = $78,000
Profit: $26,000

Profit change: ($26,000 - $40,000) / $40,000 = -35%
```

**Conclusion:** Revenue up 4%, but profit down 35%!

#### Break-Even Volume Increase

Formula para profit neutral:

```
Required Volume Increase % = Discount % / (Margin % - Discount %)
```

**Example:**
- Original margin: 40% ($100 price, $60 cost)
- Discount: 20%

```
Required Increase = 20% / (40% - 20%) = 20% / 20% = 100%
```

**Meaning:** Need to DOUBLE volume just to maintain same profit.

## 11. Price Changes

### 11.1 When to Increase Prices

#### Triggers

✅ **Cost increases**
- Raw materials up
- Labor costs up
- Overhead up

✅ **Value improvements**
- New features
- Better quality
- Enhanced service

✅ **Market conditions**
- Supply shortage
- Competitors raised prices
- High demand

✅ **Strategic**
- Reposition brand
- Improve margins
- Reduce demand to manageable level

#### How Much to Increase

**Rule of thumb:** Pass through 70-100% of cost increases.

**Example:**
```
Cost increase: 10%
Price increase: 7-10%
```

**Analysis:**
```
Old: Price $100, Cost $60, Margin 40%
Cost increases 10% to $66

Option A: No price change
New margin: ($100 - $66) / $100 = 34% (margin down 6 points)

Option B: Increase price 10% to $110
New margin: ($110 - $66) / $110 = 40% (maintains margin)

Option C: Increase price 5% to $105
New margin: ($105 - $66) / $105 = 37.1% (middle ground)
```

### 11.2 How to Communicate Changes

#### Best Practices

1. **Advance notice**
   - 30-90 days depending on industry
   - Allows customers to budget

2. **Clear communication**
   - Explain reasons (cost pressures, value improvements)
   - Be transparent

3. **Grandfathering**
   - Existing customers at old price for X months
   - Or smaller increase than new customers

4. **Value reinforcement**
   - Remind customers of benefits
   - Highlight new features/improvements

#### Example Communication

```
Dear Customer,

Effective March 1, 2025, our prices will increase by 7%.

This adjustment reflects:
- 12% increase in raw material costs
- 8% labor cost increase
- Continued investment in product quality and customer service

Current customers will maintain existing pricing through May 31, 2025.

We remain committed to delivering exceptional value.

Thank you for your continued partnership.
```

### 11.3 Monitoring Impact

#### Metrics to Track

**Customer Retention:**
```
Retention Rate = (Customers End of Period / Customers Start of Period) × 100
```

Target: >95% retention through price increase.

**Volume Impact:**
Track units sold before/after.

**Revenue & Profit:**
```
Revenue Impact = (New Price × New Volume) - (Old Price × Old Volume)
Profit Impact = Account for volume change and cost changes
```

#### Example Post-Increase Analysis

**6 Months After 10% Price Increase:**

```
Before:
Price: $100
Volume: 10,000 units/month
Revenue: $1M/month
Cost: $60/unit
Profit: $400k/month

After:
Price: $110
Volume: 9,200 units/month (-8%)
Revenue: $1,012,000 (+1.2%)
Cost: $66/unit
Profit: $406,800 (+1.7%)

Customer retention: 96%
```

**Conclusion:** ✅ Success. Revenue and profit up despite volume decline.

## 12. Margin Improvement Strategies

### 12.1 Cost Reduction

#### Strategies

**1. Supplier Negotiation**

**Impact:** Reduce COGS by 2-10%

**2. Process Efficiency**

- Reduce waste
- Automation

**Impact:** Reduce labor costs by 10-25%

**3. Overhead Reduction**
- Renegotiate leases
- Reduce headcount (carefully)
- Outsource non-core functions

**Impact:** Reduce overhead by 5-15%

### 12.2 Price Optimization

#### Strategies

**1. Eliminate Discounting**
- Reduce promotional frequency
- Smaller discount percentages
- Require larger volumes for discounts

**2. Increase List Prices**
- Across the board or selective
- Focus on high-value products
- Focus on inelastic products

**3. Reduce Price Concessions**
- Train sales not to discount
- Approval process for discounts
- Incentivize sales on margin, not just revenue

#### Example

```
Current:
List price: $100
Avg discount: 15%
Avg realized price: $85
Volume: 10,000 units
Revenue: $850k

Reduce avg discount to 10%:
Avg realized price: $90
Volume: 9,800 units (assume 2% loss)
Revenue: $882k (+3.8%)

Profit impact even larger (assuming 40% margin):
Old profit: $850k × 0.40 = $340k
New profit: $882k × 0.44 = $388k (+14%)
```

### Caso 1: Retail Store - Margin Analysis

**Business:** Electronics retail store

**Current P&L (Annual):**
```
Revenue: $2,000,000
COGS: $1,400,000 (70%)
────────────────────────────
Gross Profit: $600,000 (30%)

Operating Expenses:
  Rent: $120,000
  Salaries: $280,000
  Marketing: $60,000
  Utilities: $30,000
  Other: $50,000
  Total OpEx: $540,000
────────────────────────────
Net Profit: $60,000 (3%)
```

**Problem:** Net margin only 3%, vulnerable to any cost increase.

**Analysis:** Where can we improve?

**Option 1: Improve Gross Margin to 35%**

How? 
- Negotiate better supplier terms (target 3%)
- Reduce markdowns (target 2%)

```
New COGS = $2M × (1 - 0.35) = $1,300,000
Savings = $100,000

New Net Profit = $60k + $100k = $160,000 (8% margin)
```

**Option 2: Increase Prices 5%**

Assume 3% volume loss due to elasticity.

```
New Volume = 97% of current
New Revenue = $2M × 1.05 × 0.97 = $2,037,000
New COGS = $1,400k × 0.97 = $1,358,000
New Gross Profit = $679,000

Net Profit = $679k - $540k = $139,000 (6.8% margin)
```

**Option 3: Reduce OpEx 10%**

```
New OpEx = $540k × 0.90 = $486,000
Savings = $54,000

New Net Profit = $60k + $54k = $114,000 (5.7% margin)
```

**Option 4: Combination (Most Realistic)**

- Improve gross margin 2.5% (half of Option 1)
- Increase prices 2.5% (half of Option 2, with 1.5% volume loss)
- Reduce OpEx 5%

```
COGS improvement: $50k
Revenue: $2M × 1.025 × 0.985 = $2,019,250
COGS: $1,400k × 0.985 = $1,379,000
Gross Profit: $640,250

OpEx: $540k × 0.95 = $513,000

Net Profit: $640,250 - $513,000 = $127,250 (6.3% margin)
```

**Recommendation:** Pursue Option 4 (combination approach).

### Caso 3: Restaurant - Break-Even and Menu Pricing

**Restaurant:** Fast-casual Mexican

**Fixed Costs (Monthly):**
```
Rent: $8,000
Salaries (mgmt, fixed staff): $25,000
Utilities: $2,500
Insurance: $1,500
Other: $3,000
Total FC = $40,000/month
```

**Menu Analysis (Top 5 items, 80% of sales):**

| Item | Price | Food Cost | Labor (variable) | Total VC | CM | CM % |
|------|-------|-----------|------------------|----------|-----|------|
| Burrito | $9.50 | $2.80 | $1.20 | $4.00 | $5.50 | 57.9% |
| Bowl | $10.50 | $3.20 | $1.00 | $4.20 | $6.30 | 60.0% |
| Tacos (3) | $8.95 | $2.60 | $1.40 | $4.00 | $4.95 | 55.3% |
| Quesadilla | $7.95 | $2.40 | $0.80 | $3.20 | $4.75 | 59.7% |
| Salad | $9.95 | $3.00 | $0.90 | $3.90 | $6.05 | 60.8% |

**Current Mix (monthly):**
- Burrito: 2,000 (40%)
- Bowl: 1,500 (30%)
- Tacos: 800 (16%)
- Quesadilla: 400 (8%)
- Salad: 300 (6%)
- **Total: 5,000 orders/month**

**Current Performance:**

```
Weighted Avg CM:
(2,000×$5.50 + 1,500×$6.30 + 800×$4.95 + 400×$4.75 + 300×$6.05) / 5,000
= $28,875 / 5,000 = $5.78 per order

Total CM = 5,000 × $5.78 = $28,900
Fixed Costs = $40,000

Profit = $28,900 - $40,000 = -$11,100 (LOSS!)
```

**Break-Even Analysis:**

```
BEP = FC / Avg CM = $40,000 / $5.78 = 6,920 orders/month

Current = 5,000
Shortfall = 1,920 orders (38% increase needed!)
```

**Options to Reach Profitability:**

**Option A: Increase Volume to BEP**

Need 6,920 orders (38% increase).

Marketing required, new customers.  
Difficult in short term.

**Option B: Increase Prices**

Increase all items 10%:

```
New Weighted Avg Price = $9.36 × 1.10 = $10.30
New CM = $10.30 - $4.00 (VC unchanged) = $6.30

Total CM = 5,000 × $6.30 = $31,500
Profit = $31,500 - $40,000 = -$8,500

Better but still loss!
```

Need more:

```
Required Avg CM = $40,000 / 5,000 = $8.00
Current Avg VC = $4.00
Required Avg Price = $8.00 + $4.00 = $12.00

Current Avg Price = $9.36
Required Increase = ($12.00 - $9.36) / $9.36 = 28%
```

28% increase is too much (customers will leave).

**Option C: Combination**

- Increase prices 15%
- Push higher-margin items (Bowl, Salad)
- Increase volume 15%

```
New Avg Price = $9.36 × 1.15 = $10.76
New CM = $10.76 - $4.00 = $6.76
New Volume = 5,000 × 1.15 = 5,750

Total CM = 5,750 × $6.76 = $38,870
Profit = $38,870 - $40,000 = -$1,130

Close! Need small additional tweak.
```

**Option D: Best Solution (Combination + Cost Reduction)**

- Increase prices 15%
- Increase volume 15%
- Reduce fixed costs 5% (negotiate rent, optimize labor scheduling)

```
Total CM = $38,870 (from Option C)
New FC = $40,000 × 0.95 = $38,000

Profit = $38,870 - $38,000 = $870/month

Margin = $870 / ($10.76 × 5,750) = 1.4% (thin but positive!)
```

**Recommendation:** Implement Option D.

**Next steps:**
- Menu engineering to promote Bowl and Salad (higher CM%)
- Consider reducing portions slightly on high-VC items
- Monitor closely and aim for 6,000+ orders for healthier margin
